Hover identity for the nearest-point dots — a behavior-only change inside two existing blocks, no new contract fields (bundle sha `d357e0ef…f31d8e1`). Every layer registered through `registerNearestPointLayer` (police/fire stations everywhere, NYC/SF school-site, the post-office/library/early-voting amenity layers) gains it with no fork-side change.

- **`nearest-point-factory`**: the factory's circleMarkers were the one overlay surface with no hover identity — `hoverContainingLayers` only walks polygon layers, and the per-pin white tooltips were retired in favor of the shared dark snapshot, so hovering a dot named nothing. `pointToLayer` now wires each dot into the existing POI-pin path: mouseover surfaces the feature in the shared hover snapshot, anchored to the dot (the snapshot's standalone-row fallback — built for a hovered pin whose layer has no polygon row — renders it; hover-explorer's rendering needed nothing new); mouseout clears it, guarded by token identity so moving dot-to-dot never wipes a fresh hover. Identity derives from the same `line()` the card renders — the segment before the first " — " separator reads as the name, the remainder as the address sub-line — so the card and the hover popup cannot disagree, the same derivation rule the other factories follow. Touch devices keep the existing `hoverCapable` gate: a tap never pops the popup.
- **`hover-explorer`** (one-line hardening): the address sub-line renders only when the hovered pin actually carries an address — an empty span still showed its CSS `📍` prefix, and nearest-point dots can hover without one (a `line()` with no " — " separator).
- Consume-side scripts (`apply_engine.py`, `check_engine_parity.py`, `generate_metro_files.py`, `metro-worksheet.schema.json`) are unchanged.
- Fork adoption: nothing to do — the change is entirely inside the shared factory and hover machinery. A fork's bespoke point layers (outside the factory) are untouched; they can adopt the same pattern by setting the shared `hoveredPoi` token from their own marker events if they want parity.
